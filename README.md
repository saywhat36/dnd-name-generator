# dnd-name-generator

A fantasy name generator built to learn production-style Spring AI usage
with Claude Code as the development partner. Pick a race and gender, get
back names drawn from a curated list and/or generated by an LLM in a
matching style.

See [`CLAUDE.md`](./CLAUDE.md) for the project mission and conventions,
[`docs/ARCHITECTURE.md`](./docs/ARCHITECTURE.md) for the full design, and
[`docs/ROADMAP.md`](./docs/ROADMAP.md) for current status.

## Using the generator

Everything on this page is a single screen. Open the site and you land on
the name browser with Elf / Feminine / Handbook selected by default.

### Browse for names

Four rows of buttons drive the whole experience — click a button and the
list below updates immediately, no page reload:

- **Race** — Elf, Dwarf, Human, Halfling, Orc, Half-Orc, Gnome,
  Dragonborn, or Tiefling. Not every race is fully stocked yet, so some
  combinations show *"Nothing here yet — try another race, gender, or
  source."*
- **Gender** — Masculine or Feminine.
- **Source** — where the names come from:
  - **Handbook** — hand-curated names (📖).
  - **AI** — names generated by an LLM in a matching style (🤖).
  - **User submitted** — names proposed by other signed-in users and
    approved by a moderator (👤); these show a *"submitted by …"* byline.
  - **All** — every source together.
- **Sort** — Default (the source's own order), A–Z, or Z–A.

The heading shows how many names match the current selection, and each
row carries a small icon marking which source it came from.

Browsing is fully public — you don't need an account to look at names.

### Do more with an account

Register, then log in, to unlock the per-name and generation actions.
While signed out you'll see a *"Log in to favorite, report, submit, or
generate new names"* prompt instead.

- **Favourite** (♥) — save a name to your account. The heart fills in when
  it's saved; click again to remove it. Favourites persist across visits.
- **Report** (!) — flag a name you think is inappropriate. A moderator
  reviews reported names and can hide them.
- **Generate five more** — shown only on the **AI** source. Click it to
  ask the app to conjure the next batch of AI names for the race + gender
  you're viewing. Generation runs in the background — you'll see a
  *"Conjuring five more…"* indicator, and the list fills in when it's
  ready, so nothing ever blocks waiting on the model. AI names come from a
  shared pool (names others generated show up for you too), and each
  race + gender combination has a cap, after which the button disappears.
- **Suggest a name** — type a name into the *"Suggest a name"* box to
  propose it for the race + gender you're viewing. It goes into a review
  queue rather than appearing straight away; once a moderator approves it,
  it becomes visible to everyone under **User submitted** (and **All**).
  Submissions pass a basic content check and are rejected if the name
  already exists or is already pending.

### Create an account

- **Register** at `/register` with a username and password.
- **Log in** at `/login`. The link to log out (and your username) appears
  in the top-right once you're signed in.

### For moderators

Accounts with the admin role get two extra links in the top navigation:

- **Reports** — reported names, worst-offenders first, with actions to
  flag, reject, or un-flag each one.
- **Submissions** — the queue of user-proposed names, oldest first, with
  approve / reject actions (paginated, with bulk selection).

Admin access is granted out-of-band by a maintainer — there's no
self-serve promotion.

## Running locally

Requires Java 21 and Docker (for the local Postgres instance, started
automatically via Spring Boot's Docker Compose support).

```bash
./mvnw spring-boot:run
```

The session cookie is `Secure` by default, so it won't persist across
requests over plain `http://localhost`. Set `SESSION_COOKIE_SECURE=false`
in your environment (or `.env`) for local HTTP dev.
